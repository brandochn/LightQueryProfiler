# Light Query Profiler for VS Code

A powerful SQL Server and Azure SQL Database query profiler extension for Visual Studio Code. Monitor and analyze SQL queries in real-time with minimal overhead using SQL Server Extended Events.

## Features

- 🚀 **Real-time Query Profiling**: Monitor SQL Server queries as they execute
- 🔐 **Multiple Authentication Modes**: Windows Authentication, SQL Server Authentication, and Azure SQL Database
- 📊 **Detailed Metrics**: Track duration, CPU time, reads, writes, and more
- 🎯 **Cross-platform**: Works on Windows, Linux, and macOS
- 💾 **Lightweight**: Uses SQL Server Extended Events for minimal performance impact
- 🔍 **Query Details**: View full query text and execution details

## Prerequisites

- Visual Studio Code 1.85.0 or higher
- .NET 10 SDK or Runtime installed
- SQL Server 2016+ or Azure SQL Database
- Appropriate permissions for Extended Events on the target database

## Quick Start

1. **Open the Profiler View**
   - Click on the Light Query Profiler icon in the Activity Bar
   - Or run command: `Light Query Profiler: Show SQL Profiler`

2. **Configure Connection**
   - Select authentication mode (Windows Auth, SQL Server Auth, or Azure SQL)
   - Enter server address (e.g., `localhost` or `myserver.database.windows.net`)
   - Enter database name
   - For SQL Server/Azure SQL auth, provide username and password

3. **Start Profiling**
   - Click the **▶️ Start** button
   - Execute queries on your database
   - Watch events appear in real-time

4. **View Query Details**
   - Click on any event row to see the full query text
   - Monitor duration, CPU time, reads, and other metrics

5. **Stop Profiling**
   - Click **⏹️ Stop** when done
   - Use **⏸️ Pause** to temporarily stop collecting events
   - Click **🗑️ Clear** to remove all captured events

## Authentication Modes

### Windows Authentication (Integrated Security)
```
Server: localhost
Database: AdventureWorks
Authentication: Windows Authentication
```
No username/password required. Uses your Windows credentials.

### SQL Server Authentication
```
Server: localhost
Database: AdventureWorks
Authentication: SQL Server Authentication
Username: sa
Password: YourPassword123
```

### Azure SQL Database
```
Server: myserver.database.windows.net
Database: mydb
Authentication: Azure SQL Database
Username: sqladmin
Password: YourPassword123
```

## Required Permissions

The database user needs the following permissions to use Extended Events:

```sql
-- Grant Extended Events permissions
GRANT ALTER ANY EVENT SESSION TO [YourUser];
GRANT VIEW SERVER STATE TO [YourUser];

-- For Azure SQL Database, use database-scoped permissions
GRANT ALTER ANY DATABASE EVENT SESSION TO [YourUser];
GRANT VIEW DATABASE STATE TO [YourUser];
```

## Events Captured

The profiler captures the following SQL Server events:

- `sql_batch_completed` - Completed SQL batches
- `rpc_completed` - Completed remote procedure calls
- Duration, CPU time, logical reads, writes
- Query text, database name, application name
- Session ID, login name, timestamps

## Performance Impact

Light Query Profiler uses SQL Server Extended Events, which have minimal performance overhead compared to traditional SQL Trace. The extension:

- Filters out its own queries automatically
- Polls for events every 900ms (configurable)
- Only captures completed events
- Uses ring buffer target for efficient memory usage

## Troubleshooting

### "Server not found" Error
- Ensure .NET 10 SDK or Runtime is installed: `dotnet --version`
- Check that the server DLL is in the extension's `bin` folder

### Connection Errors
- Verify the connection string is correct
- Check firewall settings
- Ensure the user has Extended Events permissions
- For Azure SQL, verify the database name is specified

### No Events Appearing
- Verify profiling has started (green status indicator)
- Execute some SQL queries on the target database
- Check that you have permissions to view Extended Events
- Events generated by the profiler itself are filtered out

### Server Won't Start
```bash
# Verify .NET is installed
dotnet --version

# Should show: 10.0.x or higher
```

## Configuration

Currently, there are no user settings. All configuration is done through the profiler UI.

## Known Limitations

- Pause functionality is not yet implemented in the backend
- Maximum of one active session per instance
- Events are not persisted between VS Code sessions

## Development

### Building from Source

```bash
# Clone the repository
git clone https://github.com/your-repo/light-query-profiler

# Navigate to extension directory
cd vscode-extension

# Install dependencies
npm install

# Compile TypeScript
npm run compile

# Build the .NET server
cd ../src/LightQueryProfiler.JsonRpc
dotnet publish -c Release -o ../../vscode-extension/bin

# Package the extension
cd ../../vscode-extension
npm run package
```

### Running in Development

1. Open the `vscode-extension` folder in VS Code
2. Press F5 to launch Extension Development Host
3. Test the extension in the new window
